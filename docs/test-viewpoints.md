# テスト観点

対象サイト: https://kikunori1212-cpu.github.io/login-site/

## 参照した考え方

ISTQB/JSTQB Foundation Level の考え方をもとに、今回の小さなログイン画面へ適用できる粒度へ整理する。

- テストタイプ: 機能テスト、非機能テスト、回帰テスト
- ブラックボックス技法: 同値分割、境界値分析、ユースケーステスト
- 経験ベース技法: エラー推測、チェックリストベースドテスト

## 観点一覧

| ID | 観点 | 技法・分類 | 確認内容 | 自動化 |
| --- | --- | --- | --- | --- |
| VP-001 | 初期表示 | 機能テスト / チェックリスト | タイトル、見出し、ID、PASS、ログインボタンが表示される | TC-001 |
| VP-002 | 正常入力 | ユースケーステスト | ID と PASS を入力してログインボタンを押すと受付メッセージが表示される | TC-002 |
| VP-003 | ID 最小文字数 | 境界値分析 | ID が 1 文字でも入力として扱われる | TC-003 |
| VP-004 | 未入力 | エラー推測 / HTML5 validation | 未入力のまま送信した場合、ブラウザ標準検証で送信が止まる | TC-004 |
| VP-005 | パスワード秘匿 | 非機能テスト / セキュリティ基礎 | PASS 欄が password type で入力値を画面上マスクする | TC-005 |
| VP-006 | モバイル表示 | 非機能テスト / ユーザビリティ | モバイル幅でも主要ラベルとボタンが見える | TC-006 |
| VP-007 | 回帰確認 | 回帰テスト | 画面変更後も TC-001 から TC-006 が継続して通る | npm test |

## 実行コマンド

```powershell
npm.cmd install
npm.cmd test
```

ブラウザを表示して確認する場合:

```powershell
npm.cmd run test:headed
```
